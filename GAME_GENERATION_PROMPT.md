## CREATIVITY MANDATE — the most important section

**Every game must feel like it was designed by the world's most imaginative children's game
studio, not generated by an AI filling in a template.**

### Why creativity matters here
A child describes their dream game with 5–10 words. They are handing you their imagination.
Your job is to take that seed and grow it into something that makes them gasp with delight.
The world should feel ALIVE, surprising, and full of personality — even beyond what they asked for.

### Creative rules (apply to every single game)

**1. Make the world tell a story.**
Don't just place objects in space. The background should hint at a world that exists beyond
the screen. A space game has distant alien ships crossing in the far background. A forest has
fireflies drifting up at dusk. A candy world has tiny ants carrying sprinkles. These details
cost 5 lines of code and make the game feel like a real place.

**2. Give the environment personality.**
The sun should bob slightly. Clouds should drift at different speeds. Hills should be layered,
not flat. If you're in space, a giant ringed planet slowly rotates in the distance.
If you're underwater, caustic light shimmers on the seafloor. Never a static background.

**3. Make unexpected creative choices.**
If the child says "bunny," don't make a generic meadow. Make a MAGICAL meadow where giant
mushrooms bob in the breeze and rainbow-colored butterflies flutter in the background.
If they say "space cowboy," the ground should be red Mars dust, not generic grey.
Add details that weren't requested but are OBVIOUSLY right for the world.

**4. Character must have presence.**
The hero must feel like the star of the game. It should be noticeably larger than obstacles
(min 15% of screen height). It should have an idle animation even when not moving.
It should react to the world — tilt into jumps, squash on landing, emit particles when running.

**5. Color tells the mood.**
Pick a 4–5 color palette that is BOLD and consistent. Not random colors — a curated palette.
Space = deep indigo + electric blue + warm orange (character) + golden stars.
Candy = hot pink + lemon yellow + mint green + white sparkles.
Forest = rich emerald + warm amber + cream white + firefly gold.
Never use grey as a primary color in a children's game.

**6. The background is worth 40% of the visual score.**
Background quality is equal to character quality. A stunning character in a flat background
is a failed game. A mediocre character in a rich, layered, animated world still feels magical.
See section "3. Background" below for mandatory requirements.

**7. Surprise the child with one delightful unexpected element.**
Every game should have one thing the child didn't ask for but will love:
- A shooting star that occasionally crosses the background
- A tiny NPC character wandering in the far background
- A sun with a smile that winks occasionally  
- Collectibles that leave a sparkle trail
- The ground having little animated flowers or cracks
This is what separates a memorable game from a forgettable one.

### 5 Archetypes — implement exactly one

#### 1. free-flight-rescue
```
Camera    : Side orthographic or slight 3/4. Hero fixed near center.
Control   : Arrow keys / WASD / joystick for full 8-direction flight.
            Spacebar / tap button fires a projectile forward.
Objective : Collect/rescue a fixed number of trapped collectibles (e.g. baby dragons).
            Show "X / 10 rescued" counter prominently.
Hazards   : Storm clouds / enemy ships / storm patches drifting in from edges.
            Projectile transforms/destroys them in a sparkle burst.
Win       : All collectibles rescued. Trigger win_story screen.
Juice     : Rainbow/sparkle trail behind hero. Sparkle burst on rescue.
            Rescued collectibles float in a sparkle circle in the corner.
Example   : Dragon Sparkle Rescue — dragon flies freely, fires rainbow at clouds,
            collects glowing baby dragon eggs, wins when all 10 rescued.
```

#### 2. magic-cleanse
```
Camera    : Side-on. Hero stays on left third of screen.
Control   : Up/down arrow / touch drag to aim. Space / tap to fire.
Objective : Destroy / transform all waves of enemies / clouds / monsters.
            Enemies spawn in waves; show "Wave X of Y".
Hazards   : Enemies that shoot back or rush the player.
Win       : Clear all waves. Show win_story screen.
Juice     : Hit = enemy pops with color burst. Miss = shield flash.
            Combo counter: hitting 3+ in a row triggers "COMBO!" sparkle.
```

#### 3. collector-adventure (default runner)
```
Camera    : Side-scrolling, player on left quarter.
Control   : Space / tap to jump. Double-jump allowed.
Objective : Collect a target number of items (15 by default).
            Show score top-left with collectible icon.
Hazards   : Obstacles scroll from right; speed increases over time.
Win       : Collect target OR survive 90 seconds. Show win_story screen.
Juice     : Squash on land (scaleY 0.6 → 1.0 in 0.15s).
            Stretch on jump (scaleY 1.4 in 0.1s). Dust puff on land.
```

#### 4. lane-dodge
```
Camera    : Behind player (3D corridor) or top-down.
Control   : Left/right arrow or swipe to switch lanes (3 lanes).
Objective : Survive for 60 seconds or reach a finish line.
Hazards   : Obstacles in lanes scroll toward player.
            Bonus pickups give score or temporary shield.
Win       : Reach finish / survive time. Show win_story screen.
Juice     : Lane switch = quick tilt animation + whoosh sound.
            Near miss = camera shake + "CLOSE!" text popup.
```

#### 5. platform-jumper
```
Camera    : Side-scrolling upward or rightward.
Control   : Space / tap to jump. Coyote time 0.18s. Jump buffer 0.18s.
Objective : Reach the goal / flag / star at the top or end of level.
            Collect bonus items along the way.
Hazards   : Gaps (fall = lose life), moving platforms, enemy patrollers.
Win       : Reach goal. Show win_story screen.
Juice     : Land on platform = dust cloud. Collect = coin sparkle arc.
            Fall = dramatic downward camera pan then respawn.
```

### Juice and toy-feel (apply to ALL archetypes)

- **Squash & stretch**: every jump, land, collect, and hit must animate the character's scale.
- **Sparkle bursts**: collect = 12 colored particles exploding outward + bright flash.
- **Screen shake**: hit = 0.3s offset camera. Big win = 0.5s celebration shake.
- **Sound**: every action needs a distinct sound (collect = rising chirp, hit = thud, jump = whoosh, win = arpeggio).
- **Positive language**: "Try Again ✨" not "GAME OVER". "AMAZING!" not just score.

### Difficulty curve (apply to ALL archetypes)

```
0–8 seconds  : Safe zone — no dangerous obstacles, only collectibles visible.
               Show tutorial hint ("Tap to jump!" / "Arrows to fly!").
8–30 seconds : Gentle ramp — first obstacles appear, slow and telegraphed.
30–60 seconds: Normal challenge — standard difficulty, increasing speed/count.
60+ seconds  : Finale mode — faster, more obstacles, triumphant music tempo shift.
               If player is close to win condition, make it achievable.
```

### Win story screen (MANDATORY)

Use the `win_story` field from the GameSpec block verbatim as the headline of the win screen.
Display: big emoji (🎉) + win_story text + star rating + "Play Again" button + confetti particles.

```javascript
// Generate ground texture procedurally — adapt pattern to world
function makeGroundTexture(worldType) {
  const canvas = document.createElement('canvas');
  canvas.width = 512; canvas.height = 128;
  const ctx = canvas.getContext('2d');

  if (worldType === 'grass') {
    ctx.fillStyle = '#4caf50';
    ctx.fillRect(0, 0, 512, 128);
    // grass blades
    for (let i = 0; i < 200; i++) {
      ctx.strokeStyle = `hsl(${110 + Math.random()*20}, 60%, ${30+Math.random()*20}%)`;
      ctx.lineWidth = 2;
      const x = Math.random() * 512;
      ctx.beginPath(); ctx.moveTo(x, 128); ctx.lineTo(x + (Math.random()-0.5)*8, 90 + Math.random()*20); ctx.stroke();
    }
  } else if (worldType === 'space') {
    ctx.fillStyle = '#1a0a2e';
    ctx.fillRect(0, 0, 512, 128);
    for (let i = 0; i < 80; i++) {
      ctx.fillStyle = `hsl(${200+Math.random()*60}, 80%, ${40+Math.random()*30}%)`;
      ctx.fillRect(Math.random()*512, Math.random()*128, 3+Math.random()*8, 2+Math.random()*4);
    }
  } else if (worldType === 'candy') {
    // pink and white checkerboard
    for (let x = 0; x < 16; x++) for (let y = 0; y < 4; y++) {
      ctx.fillStyle = (x+y)%2 === 0 ? '#ff80b0' : '#ffffff';
      ctx.fillRect(x*32, y*32, 32, 32);
    }
  } else {
    // default: colorful gradient
    const g = ctx.createLinearGradient(0, 0, 0, 128);
    g.addColorStop(0, '#6dbf6d'); g.addColorStop(1, '#3a8f3a');
    ctx.fillStyle = g; ctx.fillRect(0, 0, 512, 128);
  }

  const tex = new THREE.CanvasTexture(canvas);
  tex.wrapS = THREE.RepeatWrapping;
  tex.repeat.set(8, 1);
  return tex;
}
```

```javascript
// ── PNG SPRITE SYSTEM ── use for character, collectibles, and obstacles ──────
// All PNG URLs are pre-processed to have transparent backgrounds (no white background removal needed).
function createSpriteMesh(url, w, h) {
  const tex = new THREE.TextureLoader().load(url);
  tex.encoding = THREE.sRGBEncoding;
  const mat = new THREE.MeshBasicMaterial({map: tex, transparent: true, side: THREE.DoubleSide, alphaTest: 0.01});
  return new THREE.Mesh(new THREE.PlaneGeometry(w, h), mat);
}

// In the game loop, call once per frame so sprites always face the camera:
function billboardAll(camera, ...meshes) {
  meshes.forEach(m => m.quaternion.copy(camera.quaternion));
}

// Squash & stretch on state change:
function squashLand(mesh) { mesh.scale.set(1.2,0.6,1); setTimeout(()=>mesh.scale.set(1,1,1),150); }
function stretchJump(mesh) { mesh.scale.set(0.85,1.4,1); setTimeout(()=>mesh.scale.set(1,1,1),100); }
```

```javascript
function createUnicorn() {
  const g = new THREE.Group();
  const white = new THREE.MeshStandardMaterial({color:0xf0e6ff, roughness:0.4});
  const gold  = new THREE.MeshStandardMaterial({color:0xffd700, roughness:0.3, metalness:0.6});
  const black = new THREE.MeshStandardMaterial({color:0x111111});
  const pink  = new THREE.MeshStandardMaterial({color:0xff88cc, roughness:0.5});

  // Body — elongated sphere
  const body = new THREE.Mesh(new THREE.SphereGeometry(0.5,16,12), white);
  body.scale.set(1.6, 1.0, 1.0);
  body.castShadow = true;
  g.add(body);

  // Four legs
  const legGeo = new THREE.CylinderGeometry(0.09, 0.07, 0.55, 8);
  [[-0.35,-0.45,0.22],[-0.35,-0.45,-0.22],[0.35,-0.45,0.22],[0.35,-0.45,-0.22]]
    .forEach(([x,y,z]) => {
      const leg = new THREE.Mesh(legGeo, white); leg.position.set(x,y,z);
      leg.castShadow = true; g.add(leg);
      const hoof = new THREE.Mesh(new THREE.CylinderGeometry(0.1,0.09,0.1,8), pink);
      hoof.position.set(x, y-0.32, z); g.add(hoof);
    });

  // Neck + head
  const neck = new THREE.Mesh(new THREE.CylinderGeometry(0.15,0.18,0.4,10), white);
  neck.position.set(0.6, 0.3, 0); neck.rotation.z = -0.5; neck.castShadow = true; g.add(neck);

  const head = new THREE.Mesh(new THREE.SphereGeometry(0.28,14,12), white);
  head.position.set(0.9, 0.52, 0); head.castShadow = true; g.add(head);

  // Snout
  const snout = new THREE.Mesh(new THREE.SphereGeometry(0.14,10,8), white);
  snout.scale.set(1.1,0.8,1); snout.position.set(1.15, 0.44, 0); g.add(snout);

  // Eyes

  });

  // Ears
  [-0.14, 0.14].forEach(z => {
    const ear = new THREE.Mesh(new THREE.ConeGeometry(0.07,0.18,7), white);
    ear.position.set(0.84, 0.82, z); g.add(ear);

  horn.position.set(1.08, 0.88, 0);
  horn.rotation.z = -0.25;   // tilt forward
  horn.castShadow = true;

  for(let i=0;i<6;i++){

    m.position.set(0.55+i*0.08, 0.68-i*0.04, 0.0);
    g.add(m);

    t.position.set(-0.75-i*0.05, 0.1+Math.sin(i)*0.2, Math.cos(i)*0.15);

}
```

### Jump + gravity (CORRECT ground detection — copy this exactly)

```javascript
const GRAVITY     = -22;

function updatePhysics(dt) {
```

### Collision detection (CIRCULAR — never use Math.abs square hitbox)

```javascript
function dist2D(a, b) {
  const dx = a.position.x - b.position.x;
  const dy = a.position.y - b.position.y;
  return Math.sqrt(dx * dx + dy * dy);
}

// Forgiving hitbox — 40% smaller than visual size
const COLL_RADIUS = 0.55;   // visual half-size ~0.5, forgiveness makes it slightly larger
const OBS_RADIUS  = 0.45;   // 40% smaller than the 0.75 visual half-size

function checkCollisions() {
  collPool.forEach(m => {
    if (!m.userData.active || m.userData.collected) return;
    if (dist2D(hero, m) < COLL_RADIUS) {
      m.userData.collected = true;
      m.visible = false;
      m.userData.active = false;
      score++;
      updateHUD();
      burstAt(m.position.x, m.position.y, 0xffdd00);
      playCollect();
      spawnColl();
      if (score >= WIN_SCORE) showEndScreen(true);
    }
  });

  if (invincible) return;
  obsPool.forEach(m => {
    if (!m.userData.active || m.userData.hit) return;
    if (dist2D(hero, m) < OBS_RADIUS) {
      m.userData.hit = true;        // ← flag so it can't hit twice
      lives--;
      updateHUD();
      burstAt(m.position.x, m.position.y, 0xff4444);
      playHit();
      flashInvincible();
      if (lives <= 0) showEndScreen(false);
    }
  });
}
```

### Particle burst (pre-allocated pool — NEVER spawn a separate requestAnimationFrame)

```javascript
const BURST_COUNT = 24;
const bursts = Array.from({length: BURST_COUNT}, () => {
  const m = new THREE.Mesh(
    new THREE.SphereGeometry(0.07, 4, 4),
    new THREE.MeshBasicMaterial({transparent: true, opacity: 1})
  );
  m.visible = false;
  scene.add(m);
  return {mesh: m, vx: 0, vy: 0, life: 0, maxLife: 0.5};
});

function burstAt(x, y, color) {
  const free = bursts.filter(b => b.life <= 0);
  const count = Math.min(12, free.length);
  for (let i = 0; i < count; i++) {
    const b = free[i];
    const angle = (i / count) * Math.PI * 2;
    b.mesh.position.set(x, y, 0.1);
    b.mesh.material.color.setHex(color);
    b.mesh.visible = true;
    b.vx = Math.cos(angle) * 4;
    b.vy = Math.sin(angle) * 4;
    b.life = b.maxLife;
  }
}

// Called from the main game loop every frame — NOT a separate RAF
function updateParticles(dt) {
  bursts.forEach(b => {
    if (b.life <= 0) { b.mesh.visible = false; return; }
    b.life -= dt;
    b.mesh.position.x += b.vx * dt;
    b.mesh.position.y += b.vy * dt;
    b.vy -= 6 * dt;                           // gravity on particles
    b.mesh.material.opacity = b.life / b.maxLife;
    b.mesh.visible = true;
  });
}
```
